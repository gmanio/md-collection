# TypeScript 리뷰 참고 기준

`SKILL.md`의 워크플로와 함께 사용한다. 필요한 섹션만 읽어도 된다.

## 1. 타입 시스템

- **`strict` 가정**: `strictNullChecks`, `noImplicitAny`, `strictFunctionTypes`가 켜져 있다고 보고, 그에 어긋나는 우회를 지적한다.
- **`any` / `unknown`**: 외부 데이터는 `unknown` + 가드/스키마 검증 선호. `any`는 근거와 범위를 요구한다.
- **단언(`as`, `!`)**: 더 좁은 타입으로 좁히는지, 런타임과 불일치하지 않는지 본다.
- **제네릭**: 제약 조건이 과하거나 부족하지 않은지, 공변/반공변 이슈가 없는지 확인한다.
- **`readonly` / 불변성**: 공유 상태·props에서 의도치 않은 변이 가능성을 본다.

## 2. 제어 흐름과 데이터

- **Narrowing**: `typeof`/`in`/사용자 정의 타입 가드가 일관적인지, 빈 문자열·0·`NaN` 같은 엣지를 놓치지 않았는지 본다.
- **Exhaustiveness**: `switch`와 판별 유니온에서 `default`로 `never` 검사 패턴이 가능한지 검토한다.
- **배열/객체**: optional 체이닝·null 병합이 필요한 곳인지, 인덱스 접근이 안전한지 본다.

## 3. 비동기·에러

- **Promise**: `await` 누락, `Promise`를 값으로 착각한 반환, 불필요한 직렬화 등을 본다.
- **에러 처리**: `catch`에서 삼킴 금지(로깅/재던지기/Result 패턴). 비동기 경계에서의 에러 타입을 본다.
- **취소/타임아웃**: 장시간 작업에 대한 `AbortSignal` 또는 동등한 취소 전략이 있는지 검토한다.

## 4. 모듈·API 경계

- **export**: 의도치 않은 넓은 export, 내부 타입의 외부 노출, 순환 의존 위험을 본다.
- **호환성**: 공개 함수 시그니처 변경이 호출부·타입 소비자에게 주는 영향을 명시한다.
- **런타임 경계**: JSON, query string, env, IPC 등 **런타임 값**이 타입과 맞는지 본다.

## 5. React(TSX)일 때

- **훅 규칙**: 조건문/반복문 내 훅 사용 금지 등 규칙 위반을 본다.
- **의존성 배열**: `useEffect`/`useMemo`/`useCallback`의 누락·과잉 의존성으로 인한 버그 가능성을 본다.
- **이벤트/클로저**: stale closure, 비동기 후 `setState` on unmount 등을 본다.

## 6. 보안·프라이버시

- 사용자 입력을 문자열 결합·쿼리·HTML에 넣는지(XSS/SQLi 등).
- 토큰·키·PII 로그 출력 여부.
- 권한 검사가 **서버/신뢰 경계**에서도 필요한지(클라만 검증하지 않았는지).

## 7. 성능(필요 시만)

- 불필요한 매 렌더 재생성(객체/함수/대형 배열), O(n²) 루프, 동기 I/O 등을 짚는다.
- 성능 주장은 측정·근거 없이 하지 않는다.

## 8. 테스트

- 변경된 분기·에러 경로·타입으로는 잡히지 않는 런타임 계약이 테스트로 보강됐는지 본다.
- 모킹이 실제 구현과 어긋나 **거짓 통과**를 만들지 않는지 본다.
